Two fixes make everyday runs safer.

## Relative db-dir with git include steps

A git include step clones into an absolute temp dir, so the ledger has to store a path that resolves back later. Path handling is now absolute safe end to end: the stored file path is derived from absolute forms of both the db dir and the clone, and every reconstruction that reads a file back joins the db dir only when the stored path is relative. A relative `--db-dir` such as `./database` now works with git include steps instead of failing with a file missing on disk.

## Destroy drops the ledger and skips extension owned objects

`destroy` now leaves the target in a truly clean state.

- Objects owned by an extension, such as the functions `pgcrypto` installs, are excluded from the plan so the individual drops do not fail.
- The `samna_migrate` ledger is always dropped with `DROP SCHEMA samna_migrate CASCADE`, so a following `up` re-applies from scratch.
- `--extensions` adds `DROP EXTENSION <name> CASCADE` for every extension the tree creates. `plpgsql` is never dropped.
